# Cooking Mode for YouTube

Chrome MV3 extension that turns YouTube cooking videos into a readable recipe side panel.

## What This Builds

Cooking Mode adds a `Cook` button on likely cooking videos and Shorts. When clicked, it opens a side panel with a clean recipe card: overview, time/servings, equipment, grouped ingredients, grouped instructions, notes, source, copy button, and wake-lock controls.

It is built for the messy reality of YouTube cooking videos:

- If the description links to a recipe website, the backend fetches that page and extracts recipe data.
- If no recipe page exists, the backend asks a local AI model to create a recipe from title, description, and captions.
- If AI/backend is unavailable, the extension falls back to its bundled local parser/model.
- The screen can stay awake until the video ends or for a chosen time.

## Architecture

- `src/contentScript.ts`: injects YouTube button, reads video title/description/captions, handles wake lock.
- `src/background.ts`: opens side panel, stores recipe state, calls local backend agent.
- `src/sidepanel.ts`: renders the recipe card and settings UI.
- `scripts/recipe-agent.mjs`: local backend agent for linked recipe scraping and AI provider calls.
- `src/localRecipeModel.ts`: bundled fallback recipe model for offline-ish extraction.
- `src/recipeParser.ts`: heuristic parser/detector.

## AI Providers

Default:

- Local Ollama, model `llama3.2:3b`

Optional API providers:

- OpenAI API
- Groq
- DeepSeek
- Gemini
- Claude

ChatGPT Plus/Pro subscriptions cannot be used as API credentials. OpenAI bills ChatGPT and API access separately, so use an OpenAI API key for OpenAI-backed extraction.

## Setup

```sh
npm install
npm run build
```

For local AI:

```sh
npm run agent:pull-model
npm run agent
```

Load the extension:

1. Open `chrome://extensions`
2. Enable Developer mode
3. Click `Load unpacked`
4. Choose `dist/`

## Usage

1. Start backend with `npm run agent`
2. Open a YouTube cooking video
3. Press `Cook`
4. Use side panel settings to choose provider/API key/model

## Shipping Notes

The checked-in source is the app. `dist/` is generated by `npm run build` and is the folder loaded into Chrome.

Do not commit:

- `node_modules/`
- `.context/`
- local secrets/API keys
